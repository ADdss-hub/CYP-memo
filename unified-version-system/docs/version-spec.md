# 通用版本规范

## 版本格式

采用语义化版本控制规范 (SemVer)，版本号格式为 `vMAJOR.MINOR.PATCH`：

- **MAJOR** 主版本号：当你做了不兼容的API修改时递增
- **MINOR** 次版本号：当你做了向下兼容的功能性新增时递增
- **PATCH** 修订号：当你做了向下兼容的问题修正时递增

额外标签标识：
- `-alpha.N` Alpha版本
- `-beta.N` Beta版本
- `-rc.N` 候选版本

示例：
- v1.0.0
- v2.1.3-beta.1
- v3.0.0-alpha.2

## 版本变更规则

### PATCH 版本号递增
- 向下兼容的缺陷修复
- 内部优化和重构
- 文档更新

### MINOR 版本号递增
- 向下兼容的新功能
- 弃用现有功能（但保留向后兼容）
- 大量新特性添加

### MAJOR 版本号递增
- 不兼容的API修改
- 功能删除
- 重大架构调整

## 版本发布原则

1. 稳定版本不能跟随预发布版本
2. 预发布版本可以跟随稳定版本
3. 版本号必须是数字，禁止使用字母等其他字符
4. 版本号递增必须是单调递增的
